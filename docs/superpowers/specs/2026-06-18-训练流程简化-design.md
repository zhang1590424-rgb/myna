# 训练流程简化设计

## 背景

当前 Web 控制台把「准备环境」「选方向」「喂数据」拆成连续步骤。实际使用时，「准备环境」没有明确列出依赖项，模型下载藏在确认页的高级设置里；「选方向」单独成步，但用户选完以后感知不到直接收益。

本次调整聚焦首轮小白体验：用户先确认本机能不能训练，再上传数据，不在中间回答抽象的分类题。

## 目标

1. 准备环境页列出训练前依赖：Python、LLaMA-Factory、PyTorch MPS、本地模型。
2. 模型准备放回准备环境页，支持在这里下载和选择可用模型。
3. 取消独立「选方向」步骤，把训练类型降级为训练数据页里的辅助选项。
4. 保留训练档位、高级参数、loss 曲线和导出能力，但不让它们抢主线。

## 新流程

| 步骤 | 名称 | 主任务 |
|---|---|---|
| 0 | 准备环境 | 看依赖是否就绪；下载或选择本地模型 |
| 1 | 训练数据 | 选择数据类型，上传并校验数据 |
| 2 | 确认设置 | 确认模型、数据、训练档位；高级参数默认折叠 |
| 3 | 开始训练 | 查看训练进度；loss 曲线默认折叠 |
| 4 | 试用对比 | 对比训练前后回答；导出 LoRA 或完整模型 |

## 训练类型的职责

训练类型不再是流程步骤，只是帮助系统解释数据。

| 类型 | 影响 |
|---|---|
| 客服问答 | 示例数据、上传提示、默认 system prompt、默认对比问题 |
| 知识问答 | 示例数据、上传提示、默认 system prompt、默认对比问题 |
| 角色语气 | 示例数据、上传提示、默认 system prompt、默认对比问题 |
| 文本改写 | 示例数据、上传提示、默认 system prompt、默认对比问题 |
| 不确定 | 使用通用模板，允许用户先跑通 |

模型不由训练类型决定。模型属于准备环境，默认推荐最稳的小模型。

## 参数和训练指标

- 训练档位保留在确认设置页，默认可见：快速、标准、精细。
- 轮数、学习率、LoRA 秩、批大小保留在高级设置中，默认折叠。
- loss 曲线保留在训练页，默认折叠，用于排障和进阶判断。
- 训练结果仍以试用对比为主，不让专业指标成为小白判断标准。

## 用户影响

用户看到的主线变成：环境准备好了吗、数据能不能训练、确认后开始、最后看效果。训练类型仍然存在，但变成一个轻量选择，不再阻断流程。

## 验证

实现后运行：

```bash
.venv/bin/python -m unittest discover -s tests
.venv/bin/python -m ruff check local_trainer tests
```

并用浏览器检查 `http://127.0.0.1:4180`：

- 左侧步骤为 0 到 4。
- 准备环境页能看到依赖状态和模型下载入口。
- 训练数据页能选择数据类型并上传文件。
- 确认设置页保留训练档位和折叠高级参数。
- 开始训练页保留折叠 loss 曲线。
