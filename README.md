# Myna 本地模型微调工具

Myna 是一个帮产品经理、运营和个人开发者亲手体验模型后训练的本地工具。你准备一批示例数据，在自己的 Mac 上跑一轮 LoRA SFT 训练，然后直接对比训练前后的回答差异。

**Myna 的目标不是训练生产可用模型，而是帮你建立后训练直觉：数据长什么样、训练怎么跑、参数会影响什么、效果变化怎么判断。**

---

## 一、Myna 是什么

### 它解决什么问题

大模型后训练对非技术用户通常是黑箱：听得懂“微调”“SFT”“LoRA”，但没有亲手跑过，很难判断一个想法到底需要什么数据、多少数据、会不会有效。

Myna 把这个过程压缩成一个本机 Web 控制台：

- **从首页进入工作流**：首页会展示最近训练、最近测评和三个常用入口，不必记页面顺序。
- **亲身体验完整流程**：准备数据、点击训练、查看进度、对比效果。
- **建立直觉**：理解训练数据、参数档位、loss 下降和效果变化的关系。
- **快速验证想法**：例如“教 AI 学客服话术可不可行”，先用小模型几分钟跑一轮。
- **降低沟通成本**：下次和算法同学讨论后训练方案时，你知道他们在说什么。

### 核心体验

你给 Myna 一批“问题 -> 正确答案”的示例，它在你的 Mac 上调用 LLaMA-Factory 做真实训练。训练完成后，你用同一个问题同时看训练前和训练后的回答。看到差异的那一刻，就能直观理解“微调”到底改变了什么。

### 30 秒看懂核心概念

不用先背概念才能使用。Myna 界面里的专业选项都有中文说明，默认值已经预填。

| 术语 | 人话翻译 | 类比 |
|---|---|---|
| 底座模型 | 出厂状态的 AI，什么都懂一点但不专精 | 刚毕业的大学生 |
| 后训练（微调） | 用你的数据教模型变成某类任务的专家 | 岗前培训 |
| SFT（监督微调） | 给一批“问题 -> 标准答案”的示例，让模型照着学 | 标准话术手册 |
| DPO（偏好优化） | 给同一个问题的“更好答案”和“更差答案”，让模型学偏好 | 主管点评回答好坏 |
| 训练数据 | 你喂给模型学习的示例 | 培训教材 |
| LoRA | 一种省资源训练方式，只改模型的一小部分 | 不重建大楼，只装修一个房间 |
| 训练轮数（Epoch） | 让模型把数据反复看几遍 | 复习课本的遍数 |
| Loss（损失值） | 模型答错的程度，通常越低越好 | 考试错误率 |
| 实验 | 一次完整训练过程 | 一轮培训项目 |

## 二、安装与启动

Myna 运行在你的 Mac 本地，不需要注册账号，不需要云服务器。安装和下载模型需要联网，训练过程在本机完成。

### 系统要求

| 项目 | 要求 |
|---|---|
| 电脑 | macOS，推荐 Apple Silicon M1/M2/M3/M4；Intel Mac 可跑但会明显变慢 |
| Python | 3.10+ |
| Git | 已安装 |
| 内存 | 建议 16 GB 以上 |
| 磁盘空间 | 至少 8 GB 空余，建议 20 GB 以上 |
| 网络 | 安装依赖和下载模型时需要联网 |

如果 Python 未安装，可先执行：

```bash
brew install python@3.11
```

### 给 AI 编程助手的安装提示词

如果你不是程序员，直接把下面这段发给 Codex、Trae、Cursor、Claude Code 等 AI 编程助手：

```text
请帮我在我的 Mac 上安装并启动 Myna 这个项目：

1. 把代码克隆到本地：https://github.com/zhang1590424-rgb/myna
2. 进入项目目录，按顺序执行下面的步骤，每步执行完告诉我结果，遇到报错先帮我排查再继续：
   - scripts/install.command   安装运行环境和训练引擎
   - 安装完环境后，顺便帮我把推荐的入门模型 Qwen3.5-0.8B 下载到本地（用项目 README 里给出的下载命令，和界面走的是同一个下载方式、放到同一个位置）
   - scripts/doctor.command    检查环境是否就绪
   - scripts/start.command     启动服务
3. 启动成功后，把浏览器访问地址发给我。
```

### For AI Agents: 一键部署指南

如果你是 AI Agent，用户希望你帮他把 Myna 在本机跑起来。按以下步骤执行即可：

```bash
# 1. 克隆项目
git clone https://github.com/zhang1590424-rgb/myna.git
cd myna

# 2. 安装环境：创建 .venv、安装 Python 依赖、准备 LLaMA-Factory
bash scripts/install.command

# 3. 预下载入门模型 Qwen3.5-0.8B（约 2 GB）
# 必须用与界面相同的 ModelScope cache_dir=models，不要换成 huggingface-cli，否则界面可能识别不到
MODELSCOPE_DOWNLOAD_PARALLELS=4 .venv/bin/python -c "from modelscope import snapshot_download; snapshot_download('Qwen/Qwen3.5-0.8B', cache_dir='models', max_workers=8)"

# 4. 健康检查
bash scripts/doctor.command

# 5. 运行测试和基础 API 检查
bash scripts/verify.command

# 6. 启动服务
bash scripts/start.command
```

第 3 步若下载失败，不影响安装是否成功。可以告诉用户先跳过，之后在界面「模型」页面手动下载。

成功标志：

- `install.command` 最后输出 `[install] 安装完成`
- `doctor.command` 无 `[FAIL]` 项（`[WARN]` 可结合提示判断）
- `verify.command` 测试全部通过
- `start.command` 输出 `Uvicorn running on http://127.0.0.1:4180`
- 浏览器能打开 `http://127.0.0.1:4180`

### 日常启动

安装完成后，以后每次使用优先双击项目根目录的 **「Myna.app」**。它会像普通 macOS 应用一样打开 Myna 窗口，也可以固定到 Dock 里。

`Myna.app` 是项目内的一键启动入口，不是独立安装包。它依赖当前项目目录里的 `.venv`、`scripts/`、`local_trainer/` 和 `web/`，所以换到新电脑后仍需先完成安装步骤。若需要排查页面兼容问题，菜单栏仍提供 **Myna → 在浏览器中打开**。

如果 macOS 首次拦截这个应用，在 `Myna.app` 上右键，选择「打开」，再确认一次即可。旧入口 **「启动Myna.command」** 仍然保留，无法打开应用时可以双击它启动。

点窗口左上角红色关闭按钮只会收起窗口，不会停止 Myna，适合训练过程中临时收起界面。用完后在 Dock 图标上右键选择「退出」，或在菜单栏选择 **Myna → 退出 Myna**。退出后，下载的模型、上传的数据和训练记录都会保留在本机。

### 升级到最新版

Myna 持续迭代。当作者发布了新功能，你可以随时升级到最新版本。

**升级不会动你的数据**：下载好的模型、上传的数据集和训练记录分别保存在 `models/`、`runtime/` 目录里，升级只更新程序代码，不会清空它们。

**最简单的方式**：双击项目根目录的 `scripts/update.command`。它会自动完成「拉取最新代码 → 更新依赖 → 按需同步训练引擎」，最后提示你重启即可。升级完建议再双击 `scripts/doctor.command` 确认环境正常。

如果你更习惯交给 AI 编程助手，也可以把项目目录发给它并说：

```text
请帮我把 Myna 升级到最新版本：
1. 在项目目录里运行 scripts/update.command
2. 升级完运行 scripts/doctor.command 确认环境正常
3. 过程中遇到报错先帮我排查再继续，完成后告诉我结果
```

手动升级（给熟悉命令行的用户）：

```bash
git pull                                              # 拉取最新代码
.venv/bin/python -m pip install -r requirements.txt   # 更新依赖
# 仅当 scripts/install.command 里的 LLAMA_FACTORY_REF 有变化时，才需重装训练引擎
scripts/start.command                                 # 重启服务
```

> 提示：如果升级时提示「检测到本地有未提交的改动」，通常是你或工具改过项目内文件。把改动备份后再升级，或让 AI 助手帮你处理。

## 三、使用流程

打开 Myna 后会先进入「首页」。首页提供「新建实验」「准备数据」「新建测评」三个常用入口，并展示最近训练和最近测评。首次使用时，仍按下面五步走最稳：

首页会优先加载训练列表，让工作台尽快可用；测评历史会在首页出现后自动补齐，不影响你点击新建实验、准备数据或进入测评。

```mermaid
flowchart LR
    A["1. 下载模型"] --> B["2. 准备数据"]
    B --> C["3. 创建实验"]
    C --> D["4. 等待训练"]
    D --> E["5. 对比效果"]
```

### 第 1 步：下载模型

进入「模型」页面，选择一个底座模型点击「下载」。

第一次建议选 **Qwen3.5-0.8B**（约 2 GB）：下载快、训练快，适合先跑通全流程。确认方向有效后，再尝试 2B 模型。

### 第 2 步：准备数据

进入「数据」页面，点击「上传数据」。

1. 选数据类型：大多数场景选「问答数据（SFT）」。
2. 上传文件：支持 CSV、JSON、JSONL、XLSX。文件里至少有两类信息：问题和回答。

没有数据也能体验。页面底部提供客服话术、角色扮演、文本润色、知识问答、偏好对等示例数据，可先下载示例跑通。

### 第 3 步：创建实验

进入「实验」页面，点击「新建实验」，依次选择：

1. 底座模型：选一个已下载的模型。
2. 训练数据：选刚才上传的数据集。
3. 训练方法：首次建议选 SFT。
4. 参数档位：从快速、标准、精细三档里选一个。

| 档位 | 特点 | 建议 |
|---|---|---|
| 快速 | 最快跑完，轮数少 | 第一次试跑 |
| 标准 | 速度与效果平衡 | 大多数情况 |
| 精细 | 学得更充分，耗时更长 | 数据量较大或想进一步调效果 |
| 高级参数 | 训练轮数、学习率、LoRA rank、批大小等 | 新手无需修改 |

### 第 4 步：等待训练完成

点击「开始训练」后，实验详情页会显示：

- 进度条：当前训练进度和状态。
- 训练基本数据：本次实验使用的模型、数据、方法和关键参数。
- 训练效果：训练 loss 曲线；当 SFT 数据量达到 30 条以上时，系统会自动切出一小部分验证集，并显示验证 loss 作为“是否可能训过头”的参考。

时间参考：80 条数据 + Qwen3.5-0.8B + 标准档位，在 Apple Silicon Mac 上通常约 5-15 分钟。实际耗时受机器、模型、数据量和参数影响。

训练期间可以切到其他页面，不影响进度；也可以随时停止。

### 第 5 步：对比效果

训练完成后，进入「测评」页面。测评结果会按实验保存，刷新页面或重启服务后仍可回看。

测评支持三种方式：

| 方式 | 适合做什么 |
|---|---|
| 单问对比 | 输入一个关键问题，并排查看训练前（Base）和训练后（Fine-tuned）的回答 |
| 批量测评 | 从训练数据抽样，或手动粘贴多条问题，集中看整体变化 |
| 自由对话 | 持续和训练后的模型聊天，观察语气、知识和稳定性 |

单问和批量测评会进入历史记录；自由对话结束测评后会保存对话记录。自由对话会把模型加载到内存里，用完建议点「结束测评」或「释放模型」。

如果效果不好，优先调整数据质量和数据量，再考虑换参数或换更大的模型。

## 四、推荐体验场景

| 场景 | 适合验证什么 |
|---|---|
| 客服话术训练 | 模型能否学会固定口吻、处理方式和结束语 |
| 角色扮演 / 人设定义 | 模型能否学会稳定的人设、语气和表达习惯 |
| 文本改写 / 风格转换 | 模型能否按固定格式或规则改写文本 |
| 专属知识问答 | 模型能否记住底座模型原本不知道的产品事实 |
| 偏好对（DPO） | 模型能否偏向更简洁、更符合偏好的答案 |

这些场景的目的不是训练出可直接上线的模型，而是让你亲手感受“数据格式 -> 训练过程 -> 效果变化”的完整链路。

## 五、常见问题

### Q：需要什么配置的电脑？

推荐 Apple Silicon Mac，内存 16 GB 以上。Intel Mac 可以走通流程，但训练会明显变慢。Windows/Linux 暂不作为当前分发目标。

### Q：训练需要联网吗？

下载依赖和模型时需要联网。模型从国内 ModelScope 源下载。训练过程在本机执行，数据不会上传到云端。

### Q：数据需要多少条？

最少 3 条可以通过系统校验并启动训练，但真实效果通常需要更多数据。建议 50-200 条起步看效果。数据质量比数量重要，少量高质量标注往往比大量粗糙数据更有价值。

### Q：训练一次要多久？

取决于数据量、模型大小、机器配置和参数档位。典型参考：

| 数据量 | 模型 | 档位 | 大约耗时 |
|---|---|---|---|
| 80 条 | Qwen3.5-0.8B | 快速 | 3-5 分钟 |
| 80 条 | Qwen3.5-0.8B | 标准 | 8-15 分钟 |
| 200 条 | Qwen3.5-2B | 标准 | 30-60 分钟 |

### Q：训练完的模型能上线用吗？

不建议直接用于生产环境。Myna 用的是轻量小模型和少量数据，足以体验微调原理，但不满足生产级需求。生产训练通常需要更大模型、更多数据、GPU 集群、评测体系和工程化部署。

### Q：效果不好怎么办？

按这个顺序排查：

1. 数据质量：答案是否真的符合你想教给模型的标准。
2. 数据量：低于 30 条时，模型可能学不够。
3. 训练档位：从快速换到标准或精细。
4. 底座模型：从 0.8B 换到 2B，容量更大，可能学得更稳。

### Q：loss 和验证 loss 怎么看？

训练 loss 下降通常说明模型正在贴合训练数据，但不等于真实效果一定变好。SFT 数据量达到 30 条以上时，Myna 会留出约 15% 数据做验证集：如果训练 loss 下降而验证 loss 上升，可能是模型开始背答案。最终仍以「测评」里的训练前后回答对比为准。

### Q：测评历史会丢吗？

不会。单问、批量测评和自由对话记录会保存在本机 `runtime/workbench.db`。删除项目或清理 `runtime/` 会一起删除这些记录。

### Q：会不会把我的电脑搞坏？

不会。Myna 的文件都在项目目录内，不修改系统设置。训练时会占用较多内存和计算资源，训练结束后释放。如需彻底清理，删除项目文件夹即可。

## 六、本地数据说明

| 目录 | 内容 | 可否删除 |
|---|---|---|
| `models/` | 下载的模型缓存 | 可删，再用时重新下载 |
| `runtime/datasets/` | 上传的数据集 | 可删 |
| `runtime/runs/` | 训练任务产物；训练完成后会自动清理中间 checkpoint，保留最终 LoRA 产物 | 可删 |
| `runtime/workbench.db` | 实验元数据和测评历史 | 可删 |
| `LLaMA-Factory/` | 第三方训练引擎 | 可删，重新运行安装脚本会恢复 |
| `.venv/` | Python 虚拟环境 | 可删，重新运行安装脚本会恢复 |

所有训练数据、下载模型和训练产物都保存在用户本机。

## 七、技术架构

```text
浏览器 (127.0.0.1:4180)
    |
    v
Python FastAPI 后端 (local_trainer/)
    |
    v
LLaMA-Factory 训练引擎
    |
    v
Apple MPS / CUDA / CPU
```

当前能力：

- 本机 Web 控制台和默认首页
- 模型下载（ModelScope 国内源）
- 数据上传与校验（CSV / JSON / JSONL / XLSX）
- 真实 LoRA SFT / DPO 训练
- 实时训练进度、训练 loss 和 SFT 验证 loss 展示
- 训练前后单问对比、批量测评、自由对话
- 测评历史持久化，支持按实验回看和删除
- 多实验参数和 loss 对比
- LoRA adapter / 合并完整模型导出
- 升级脚本 `scripts/update.command`，升级代码时保留本地模型、数据和训练记录
- 环境未就绪时禁止发起训练，不提供演示或模拟训练
