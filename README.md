# Myna 本地模型微调工具

Myna 帮产品经理、运营和个人开发者在自己的 Mac 上体验模型后训练。准备一批示例数据，跑一轮 LoRA SFT 训练，对比训练前后的回答差异——建立「数据 → 训练 → 效果」的直觉，而不是训练生产可用模型。

---

## 安装与启动

运行在本地，不需要注册账号和云服务器。安装和下载模型需要联网，训练在本机完成。

### 系统要求

- macOS，推荐 Apple Silicon（M1/M2/M3/M4）；Intel Mac 可跑但明显慢
- Python 3.10+、Git
- 内存 16 GB 以上
- 磁盘至少 8 GB 空余，建议 20 GB+
- 安装和下载模型时需联网

### For AI Agents: 一键部署指南

```bash
# 1. 克隆项目
git clone https://github.com/zhang1590424-rgb/myna.git
cd myna

# 2. 安装环境
bash scripts/install.command

# 3. 预下载入门模型 Qwen3.5-0.8B（约 2 GB）
# 必须用 ModelScope + cache_dir=models，不要换 huggingface-cli
MODELSCOPE_DOWNLOAD_PARALLELS=4 .venv/bin/python -c "from modelscope import snapshot_download; snapshot_download('Qwen/Qwen3.5-0.8B', cache_dir='models', max_workers=8)"

# 4. 健康检查
bash scripts/doctor.command

# 5. 运行测试
bash scripts/verify.command

# 6. 启动服务
bash scripts/start.command
```

第 3 步下载失败不影响安装，可之后在界面「模型」页面手动下载。

成功标志：`start.command` 输出 `Uvicorn running on http://127.0.0.1:4180`，浏览器能打开该地址。

### 日常启动

双击项目根目录的 **Myna.app**，它会在独立窗口中打开 Myna 控制台。可固定到 Dock。

- 首次被 macOS 拦截：右键 Myna.app → 打开 → 确认
- 关闭窗口只是收起界面，不停止服务；退出用 Dock 右键「退出」或菜单栏 Myna → 退出 Myna
- 退出后模型、数据、训练记录都保留在本机
- 备用入口：`启动Myna.command`

`Myna.app` 依赖项目目录里的 `.venv` 和 `scripts/`，换电脑后需先完成安装。

### 升级

双击 `scripts/update.command`，自动拉取代码、更新依赖、同步训练引擎。升级不动 `models/` 和 `runtime/` 里的数据。

升级完建议跑一次 `scripts/doctor.command` 确认环境正常。

## 使用流程

首页有「新建实验」「准备数据」「新建测评」三个入口和最近记录。首次使用按五步走：

**① 下载模型** → **② 准备数据** → **③ 创建实验** → **④ 等待训练** → **⑤ 对比效果**

### 第 1 步：下载模型

「模型」页面选一个底座模型下载。推荐先选 **Qwen3.5-0.8B**（约 2 GB），下载快训练快，跑通流程后再换大模型。

### 第 2 步：准备数据

「数据」页面上传文件。支持 CSV/JSON/JSONL/XLSX，至少包含问题和回答两列。大多数场景选「问答数据（SFT）」。

没有数据也行——页面底部有客服话术、角色扮演等示例可直接下载使用。

### 第 3 步：创建实验

「实验」页面点「新建实验」，选模型、数据、训练方法（首次选 SFT）、参数档位：

| 档位 | 特点 | 建议 |
|---|---|---|
| 试跑 | 先验证流程和口吻方向 | 第一次使用、检查数据格式和训练链路 |
| 推荐 | 平衡可见变化和过拟合风险 | 第一次正式训练 |
| 加强 | 上次测评变化不明显时再试 | 数据较多，或已经有一次基线结果 |

高级参数已预填默认值，新手无需修改。

数据少于 30 条时，训练结果只能作为体感参考，适合观察口吻或格式是否有变化，不适合判断模型已经稳定学会能力；不要一开始就选「加强」。

### 第 4 步：等待训练

点「开始训练」后实时显示进度、训练 loss 曲线。SFT 数据达 30 条以上时自动切验证集并显示验证 loss。

时间参考：80 条数据 + Qwen3.5-0.8B + 推荐档位，Apple Silicon 约 5-15 分钟。训练期间可切页面或随时停止。

### 第 5 步：对比效果

「测评」页面，支持三种方式：

| 方式 | 做什么 |
|---|---|
| 单问对比 | 一个问题，并排看训练前后回答 |
| 批量测评 | 多条问题集中看整体变化 |
| 自由对话 | 和训练后模型持续聊天 |

测评结果按实验保存，重启后可回看。自由对话会加载模型到内存，用完点「结束测评」释放。

效果不好？优先调数据质量和数量，再考虑换参数或更大模型。

## 常见问题

**Q：训练需要联网吗？**
下载模型时需要（国内 ModelScope 源）。训练完全离线，数据不上传。

**Q：数据需要多少条？**
最少 3 条可启动训练。建议 50-200 条起步看效果，质量比数量重要。

**Q：训练一次多久？**

| 数据量 | 模型 | 档位 | 大约耗时 |
|---|---|---|---|
| 80 条 | Qwen3.5-0.8B | 试跑 | 3-5 分钟 |
| 80 条 | Qwen3.5-0.8B | 推荐 | 8-15 分钟 |
| 200 条 | Qwen3.5-2B | 推荐 | 30-60 分钟 |

**Q：训练完的模型能上线吗？**
不建议。Myna 用小模型 + 少量数据体验原理，不满足生产需求。生产训练需要更大模型、更多数据和 GPU 集群。

**Q：效果不好怎么办？**
依次检查：数据质量 → 数据量（低于 30 条可能不够）→ 训练档位（试跑换推荐，或推荐换加强）→ 换更大模型。

**Q：loss 怎么看？**
训练 loss 下降说明模型在学习。如果验证 loss 上升而训练 loss 下降，可能过拟合。最终以测评对比为准。

**Q：会把电脑搞坏吗？**
不会。所有文件在项目目录内，不改系统设置。训练占资源结束后释放。彻底清理删项目文件夹即可。

## 本地数据说明

| 目录 | 内容 | 可否删除 |
|---|---|---|
| `models/` | 下载的模型 | 可删，再用时重下载 |
| `runtime/datasets/` | 上传的数据集 | 可删 |
| `runtime/runs/` | 训练产物（自动清理中间 checkpoint） | 可删 |
| `runtime/workbench.db` | 实验元数据和测评历史 | 可删 |
| `LLaMA-Factory/` | 训练引擎 | 可删，重装恢复 |
| `.venv/` | Python 虚拟环境 | 可删，重装恢复 |

所有数据保存在本机，不上传云端。

## 概念速查

| 术语 | 人话 |
|---|---|
| 底座模型 | 出厂 AI，什么都懂一点但不专精 |
| 后训练/微调 | 用你的数据教模型变专家 |
| SFT | 给"问题→标准答案"示例让模型照着学 |
| DPO | 给好/差答案对让模型学偏好 |
| LoRA | 省资源训练方式，只改模型一小部分 |
| Epoch | 模型把数据看几遍 |
| Loss | 模型答错程度，越低越好 |
